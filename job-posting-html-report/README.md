# job-posting-html-report 준비 가이드

`job-posting-html-report`는 채용 공고 URL을 읽어 공고 내용과 회사 정보를 조사하고, 지원 판단과 면접 전략에 바로 쓸 수 있는 정적 HTML 리포트를 생성하는 스킬입니다.

HTML은 Linear 스타일의 어두운 제품 대시보드 디자인을 따릅니다. 결과물은 `YYYY-MM-기업명/` 폴더 안에 생성됩니다.

## 미리 준비하기

가능하면 아래 정보를 준비하세요.

- 채용 공고 URL
- 본인의 연차, 현재 직무, 기술스택
- 중요하게 보는 기준: 성장성, 안정성, 연봉, 기술문화, 워라밸, 도메인, 재택 여부
- 피하고 싶은 조건
- 이력서 또는 자기소개서 초안

이력서와 자기소개서 같은 개인 자료는 분석에는 사용할 수 있지만, 사용자가 명시적으로 요청하지 않으면 결과 폴더에 저장하지 않습니다.

## 사용하는 방법

Codex에 `$job-posting-html-report`로 시작하는 요청을 보내세요.

```text
$job-posting-html-report 이 채용 공고를 분석해서 HTML 리포트로 만들어줘: https://example.com/jobs/frontend
```

개인화 조건을 함께 줄 수도 있습니다.

```text
$job-posting-html-report 5년차 프론트엔드 개발자 관점에서 이 공고를 분석해줘. 성장성, 기술문화, 재택 여부를 중요하게 보고, 야근이 잦은 조직은 피하고 싶어. URL: https://example.com/jobs/frontend
```

## 생성되는 파일

기본 결과물은 아래 구조입니다.

```text
YYYY-MM-기업명/
  index.html
  styles.css
  assets/
  data/
  sources/
  script.js  # 선택 파일
```

- `index.html`: 최종 HTML 리포트
- `styles.css`: Linear 스타일 기반 리포트 스타일
- `assets/`: 이미지나 보조 자산
- `data/`: 추출 텍스트, 분석 데이터, 구조화 결과
- `sources/`: 출처 목록과 근거 자료
- `script.js`: 목차 하이라이트나 접기/펼치기 같은 보조 기능

## 실행 흐름

1. 채용 공고 URL을 확인합니다.
2. 공고 본문을 추출합니다.
3. 회사명과 포지션명을 확정합니다.
4. 회사 공식 자료, 공시/재무/투자 자료, 뉴스, 채용/기술 블로그를 조사합니다.
5. 확인된 사실, 추정, 확인 불가 정보를 나눕니다.
6. 회사, 포지션, 지원자 적합도, 리스크를 분석합니다.
7. 지원 동기, 이력서 키워드, 면접 질문, 합류 전 확인 기준을 만듭니다.
8. Linear 디자인을 따른 HTML 리포트 패키지를 생성합니다.
9. 출처, 누락 정보, 디자인, 렌더링을 검증합니다.

## 리서치와 출처 원칙

출처 우선순위는 다음과 같습니다.

1. 회사 공식 자료
2. 공시, 재무, 투자 자료
3. 신뢰 가능한 뉴스
4. 채용 블로그, 기술 블로그
5. 잡플래닛, 블라인드, 커뮤니티 같은 평판성 자료

평판성 자료는 사실 근거가 아니라 참고 신호로만 사용합니다. 출처가 없거나 확인되지 않은 내용은 단정하지 않고 `확인 불가` 또는 `추가 확인 필요`로 표시합니다.

## 중단되거나 실패하는 경우

다음 상황에서는 중단하거나, 리포트 안에 실패 사유를 남깁니다.

- 채용 공고 URL이 없는 경우
- 공고 본문을 추출할 수 없는 경우
- 회사명이 모호하거나 동명이인 회사 가능성이 있는 경우
- 재무, 투자, 최근 뉴스 등 핵심 자료를 확인할 수 없는 경우
- 브라우저 렌더링 검증을 할 수 없는 환경인 경우

실패나 누락은 숨기지 않고, 지원자가 무엇을 추가로 확인해야 하는지 함께 기록합니다.
