# Technical Writing

기술 문서는 독자가 문제를 해결하고 목표를 이루도록 돕는 글이다. 정확한 정보를 빠르고 명확하게 전달하는 데 집중한다.

## 진행 방식

요청 범위를 먼저 판단한 뒤 필요한 reference만 읽어라. 전체 문서 작성, 리뷰, 재작성처럼 범위가 넓으면 `references/workflow.md`를 먼저 읽고, 그 안내에 따라 다른 reference를 추가로 읽어라.

## Reference 라우팅

- `references/workflow.md`: 문서 전체 작성, 리뷰, 재작성, 품질 개선, 새 문서 기획처럼 여러 단계가 필요한 요청에 읽어라.
- `references/document-types.md`: 문서 유형 선택, 튜토리얼/How-to/트러블슈팅/API 참조/개념 설명 문서 작성, 유형별 템플릿이 필요할 때 읽어라.
- `references/information-architecture.md`: 목차, 페이지 범위, 개요, 제목, 정보 순서, 내비게이션, 크로스링크, 문서 세트 구조를 다룰 때 읽어라.
- `references/sentence-editing.md`: 한국어 문장 교정, 간결화, 구체화, 자연스러운 표현, 능동형, 용어 일관성 점검이 필요할 때 읽어라.
- `references/prompts.md`: 사용자가 재사용 가능한 AI 프롬프트를 요청하거나, 문서 리뷰용 프롬프트를 구성할 때 읽어라.
- `references/examples.md`: 구체적인 예시, before/after 비교, 템플릿 적용 사례, 샘플 문서가 필요할 때 읽어라.

## 적용 원칙

- 먼저 독자의 목표와 현재 문서의 목적을 파악해라.
- 문서 유형, 정보 구조, 문장 품질 순서로 점검해라.
- 여러 reference가 필요하면 함께 읽어라. 전체 문서 리뷰에서 하나의 reference만 읽고 끝내지 마라.
- 결과는 실행 가능한 개선안으로 제시해라. 가능하면 수정 예시를 포함해라.
- 원칙은 강제 규칙이 아니라 권장 기준으로 적용해라. 문맥상 더 나은 선택이 있으면 이유를 설명하고 조정해라.
