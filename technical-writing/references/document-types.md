# 문서 유형

기술 문서는 독자의 목적에 따라 유형을 나눈다. 유형을 정하면 포함해야 할 정보와 구조를 쉽게 결정할 수 있다.

## 유형 선택 기준

| 유형 | 독자의 목적 | 적합한 상황 |
| --- | --- | --- |
| 학습 중심 문서 | 새로운 기술이나 도구의 흐름을 처음 익힌다 | 시작하기, 튜토리얼, 첫 연동 가이드 |
| 문제 해결 문서 | 특정 문제를 해결하거나 작업을 완료한다 | How-to, 설정 가이드, 트러블슈팅 |
| 참조 문서 | 특정 기능, API, 옵션을 빠르게 확인한다 | API 레퍼런스, 옵션 목록, 오류 코드 |
| 설명 문서 | 개념, 원리, 배경을 깊이 이해한다 | 아키텍처 설명, 도메인 지식, 동작 원리 |

유형은 항상 하나만 선택해야 하는 것은 아니다. 문서 목적이 섞여 있으면 주 목적을 정하고, 보조 목적은 별도 섹션이나 별도 문서로 분리한다.

## 학습 중심 문서

독자가 새로운 기술이나 도구를 처음부터 끝까지 막힘없이 따라 하도록 돕는다.

포함할 것:

- 명확한 학습 목표
- 사전 준비 사항과 환경 설정
- 단계별 안내
- 실제로 실행 가능한 코드 예제
- 단계별 성공 여부를 확인하는 방법
- 필요하면 FAQ나 자주 발생하는 문제

체크리스트:

- 학습 목표가 명확한가?
- 직접 따라 했을 때 오류가 생기지 않는가?
- 간단한 예제에서 점진적으로 난이도가 올라가는가?
- 모든 코드 예제가 실제로 실행 가능한가?

시작하기 문서는 주요 흐름과 필수 개념을 이해하도록 돕는다. 튜토리얼 문서는 더 구체적인 목표와 결과물을 갖고, 단계를 따라 하며 결과물을 만든다.

## 문제 해결 문서

독자가 직면한 문제를 빠르고 효과적으로 해결하도록 돕는다.

포함할 것:

- 명확한 문제 상황 또는 작업 목표
- 원인과 현상의 구분
- 바로 적용할 수 있는 해결 방법
- 코드, 명령어, 설정 예시
- 환경별 차이
- 해결 후 확인 방법

체크리스트:

- 에러 원인뿐 아니라 필요한 기본 지식도 설명했는가?
- 즉시 적용할 수 있는 해결 방법이 있는가?
- OS, 라이브러리 버전, 설정 차이를 고려했는가?
- 해결책이 어떤 원리로 문제를 해결하는지 설명했는가?

How-to 가이드는 특정 기능 구현이나 작업 완료에 초점을 둔다. 트러블슈팅 문서는 이미 발생한 문제의 진단과 해결에 초점을 둔다.

## 참조 문서

독자가 특정 기술, 도구, 시스템에 대한 정확하고 완전한 정보를 빠르게 찾도록 돕는다.

포함할 것:

- 간결한 개요와 주요 기능
- 시그니처 또는 구문
- 매개변수, 옵션, 속성
- 반환 값과 타입
- 사용 예제
- 제한사항과 주의사항
- 관련 API 또는 연계 방법

체크리스트:

- 검색과 탐색이 쉬운가?
- 정보가 정확하고 완전한가?
- 같은 종류의 항목이 일관된 구조로 작성되었는가?
- 실용적인 예제 코드가 포함되어 있는가?

참조 문서는 방대한 정보를 다루므로 주제별 섹션, 목차, 앵커, 표를 활용한다. API 키, 인증 방식, 요청 헤더처럼 사전에 알아야 할 정보는 앞부분에 둔다.

## 설명 문서

독자가 기술이나 개념의 원리, 배경, 맥락을 깊이 이해하도록 돕는다.

포함할 것:

- 개념이 등장한 배경
- 해결하려는 문제
- 기본 원리와 동작 방식
- 다른 접근 방식과의 비교
- 장단점
- 다이어그램, 흐름도, 표 같은 시각 자료
- 실제 사용 사례

체크리스트:

- 핵심 원리와 배경이 명확한가?
- 복잡한 개념을 시각화했는가?
- 선행 지식이 충분히 안내되었는가?
- 단순 사용법을 넘어 왜 그런지 설명하는가?

## 유형별 기본 템플릿

학습 중심 문서:

```markdown
# [튜토리얼 제목]

## 목표
## 사전 요구사항
## 단계별 가이드
### [첫 번째 단계]
### [두 번째 단계]
## 최종 결과 확인
```

문제 해결 문서:

```markdown
# [문제 해결 문서 제목]

## 문제 정의
## 사전 요구사항
## 해결 방법
### [첫 번째 해결 방법]
### [두 번째 해결 방법]
## 문제 해결 후 확인 방법
```

참조 문서:

```markdown
# [참조 문서 제목]

## 개요
## 속성 및 옵션
## 시그니처
## 반환 값
## 사용 예제
```

설명 문서:

```markdown
# [개념명]

## 개념 소개
## 등장 배경
## 작동 원리
## 활용
```
