# 예시

이 파일은 문서 유형과 개선 방향을 설명할 때 참고할 수 있는 축약 예시를 담는다.

## 학습 문서 before/after

개선 전:

```markdown
# React 컴포넌트 만들기

React 컴포넌트를 만드는 방법을 알아봅시다.

## 함수형 컴포넌트

함수형 컴포넌트는 다음과 같이 만듭니다.

function Welcome() {
  return <h1>Hello</h1>;
}

## Props 사용하기

props를 사용하면 컴포넌트에 데이터를 전달할 수 있습니다.
```

문제:

- 문서의 목적과 학습 목표가 명확하지 않다.
- 예제가 실행 가능한 프로젝트 맥락 없이 제시된다.
- Props 개념이 갑자기 등장해 단계적 학습이 어렵다.

개선 후 구조:

```markdown
# React로 첫 컴포넌트 만들기

이 튜토리얼에서는 React 컴포넌트를 만들고 데이터를 전달하는 방법을 배워요.

## 학습 목표
- 함수형 컴포넌트의 기본 구조를 이해해요.
- props를 사용해서 데이터를 전달하는 방법을 배워요.
- 실제로 동작하는 컴포넌트를 만들어봐요.

## 준비사항
- Node.js를 설치해요.
- 코드 에디터를 준비해요.
- HTML과 JavaScript 기본 지식이 필요해요.

## 1단계. 프로젝트 설정하기
## 2단계. 첫 컴포넌트 만들기
## 3단계. Props 추가하기
```

개선 포인트:

- 학습 목표와 준비사항을 먼저 제시한다.
- 목표, 준비, 실습, 적용, 확장 흐름으로 구성한다.
- 모든 코드 예제가 실제로 실행 가능하도록 한다.

## 트러블슈팅 문서 before/after

개선 전:

```markdown
# Hydration 오류 해결하기

Next.js에서 클라이언트와 서버의 렌더링 결과가 불일치하면 Hydration 오류가 발생할 수 있어요.

## 해결 방법

function TimeComponent() {
  const [time, setTime] = useState(0);
  useEffect(() => {
    setTime(Date.now());
  }, []);
  return <p>현재 시간: {time}</p>;
}
```

문제:

- Hydration 오류가 무엇인지 설명하지 않는다.
- 문제 상황과 해결 방법이 구조적으로 분리되지 않았다.
- 코드가 먼저 등장해 독자가 맥락을 추론해야 한다.

개선 후 구조:

```markdown
# Next.js에서 Hydration 오류를 방지하는 방법

Next.js에서 서버와 클라이언트의 렌더링 결과가 다를 때 발생하는 Hydration 오류를 방지하는 방법을 알아봅니다.

## 문제

서버에서 생성된 HTML과 클라이언트에서 렌더링된 결과가 다르면 Hydration 오류가 발생할 수 있어요.

## 해결 방법

Hydration 오류를 방지하려면 클라이언트에서만 실행해야 하는 코드를 `useEffect`로 감싸야 해요.

### 잘못된 코드
### 올바른 코드
### 참고 문서
```

개선 포인트:

- 문서 제목과 도입부에서 해결할 문제를 명확하게 말한다.
- 문제 원인, 잘못된 코드, 올바른 코드를 순서대로 제시한다.
- 독자가 적용 후 정상 동작을 확인할 수 있게 한다.
