# multi-ask 준비 가이드

`multi-ask`는 Codex의 @browser를 사용해 ChatGPT web, Gemini web, Claude web에 같은 질문을 보내고 결과를 비교하는 스킬입니다. 실행 전에 세 사이트에 로그인해 두면 중간에 멈추지 않습니다.

## 미리 로그인하기

Codex의 브라우저에서 아래 사이트를 열고 로그인하세요.

- ChatGPT web: https://chatgpt.com
- Gemini web: https://gemini.google.com
- Claude web: https://claude.ai

Claude web은 Codex 인앱 브라우저에서 Google 로그인 흐름이 실패할 수 있습니다. Claude는 `Continue with Google` 대신 이메일 로그인 방식을 사용하세요. 이메일을 입력한 뒤 Claude가 보내는 로그인 코드 또는 링크로 접속을 완료하는 방식이 가장 안정적입니다.

로그인 후 각 사이트에서 새 채팅 입력창이 보이는지 확인하세요. 유료 기능이나 딥리서치 기능을 쓰려면 해당 계정에서 그 기능이 실제로 사용 가능해야 합니다.

## 사용하는 방법

Codex에 `$multi-ask`로 시작하는 질문을 보내세요.

```text
$multi-ask 이 제품 전략의 장단점을 GPT 웹, Gemini 웹, Claude 웹 모두에게 물어보고 결론을 내려줘
```

기본 대상은 ChatGPT web, Gemini web, Claude web 전체입니다. 대상을 제한할 수도 있습니다.

```text
$multi-ask GPT 웹, Gemini 웹으로만 이 설계안을 검토해줘
```

추론 또는 사고 수준은 기본적으로 보통입니다. 높이고 싶으면 질문에 명시하세요.

```text
$multi-ask 사고를 최고로 올려서 이 아키텍처의 리스크를 분석해줘
```

딥리서치를 원하면 `딥리서치`라는 키워드를 포함하세요.

```text
$multi-ask 딥리서치로 2026년 AI 브라우저 자동화 도구 동향을 비교해줘
```

## 실행 흐름

1. Codex가 질문을 분석해 전송할 프롬프트를 제안합니다.
2. 사용자가 승인하거나 수정 요청을 합니다.
3. 승인된 뒤에만 Codex가 @browser로 선택된 AI 웹사이트를 먼저 모두 점검합니다.
4. 점검이 통과되면 GPT, Gemini, Claude 같은 선택 대상에 질문을 순서대로 모두 넣습니다.
5. Codex가 각 채팅 탭과 URL을 기억해 두고, 한 사이트에 머물러 기다리지 않고 선택 대상들을 돌면서 완료 여부를 확인합니다.
6. Codex가 모든 선택 대상의 답변을 모읍니다.
7. Codex가 각 답변의 의견과 근거를 요약하고 최종 결론을 냅니다.

## 중단되는 경우

다음 상황에서는 질문을 전송하지 않고 중단합니다.

- 선택된 사이트 중 하나라도 로그인되어 있지 않은 경우
- 선택된 사이트 중 하나라도 채팅 입력창에 접근할 수 없는 경우
- `딥리서치` 요청인데 선택된 사이트 중 하나라도 딥리서치 모드를 켤 수 없는 경우

일반 질문에서 사고/추론 설정을 찾지 못하는 경우에는 경고만 표시하고 가능한 기본값으로 진행합니다.
