# Verification Checklist

Run this checklist before claiming the HTML report is complete.

## File Checks

- `index.html` exists.
- `styles.css` exists.
- `assets/` exists.
- `data/` exists.
- `sources/` exists.
- `script.js`, if present, is not required for core reading.

## Evidence Checks

- The original job posting URL is included.
- Major company, role, growth, stability, and fit judgments include evidence or uncertainty notes.
- Confirmed information, inference, unavailable information, and items to verify are visibly separated.
- Recent news, funding, financial, and market information include dates where available.
- Reputation sources are marked as signals, not facts.

## Privacy Checks

- Resume, cover letter, or personal career drafts are not saved unless explicitly requested.
- The report does not expose unnecessary private details from user input.

## Design Checks

- Linear-style near-black canvas and charcoal panels are used.
- Accent color is restrained and consistent.
- Borders, radius, spacing, and badges feel like a compact product dashboard.
- No marketing-style hero dominates the page.
- No decorative gradient blobs or noisy visual effects are used.
- Text does not overlap, clip, or overflow panels.
- Missing information is visible, not buried.

## Rendering Checks

When possible, open `index.html` in a browser and confirm:

- Company name, position, overall judgment, recommendation, and top risks are visible in the first screen.
- Navigation reaches the main sections.
- Tables, badges, and links are readable.
- Sources section is present.
- The page remains useful with JavaScript disabled or unavailable.

If browser rendering cannot be verified, state that in the final response.
