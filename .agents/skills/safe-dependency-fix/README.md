# safe-dependency-fix

의존성/패키지 취약점을 안전하게 분석하고 수정 계획을 세우는 Codex 스킬입니다.

기본 동작은 수정이 아니라 분석입니다. 먼저 `osv-scanner`와 프로젝트별 audit 도구로 취약점을 확인하고, 직접/전이 의존성 여부와 최소 수정 후보를 한국어로 보고합니다. 실제 파일 변경은 사용자가 정확히 `수정 승인` 또는 `실제 변경 진행`이라고 말한 뒤에만 수행합니다.

## 핵심 원칙

- dry-run 단계에서는 파일을 수정하지 않습니다.
- 승인 전에는 install/update/upgrade/audit fix 계열 명령을 실행하지 않습니다.
- `osv-scanner`가 없으면 설치할지, 생태계별 audit만 수행할지 먼저 선택받습니다.
- 직접 의존성 업데이트를 우선하고, 전이 의존성은 상위 패키지 업데이트로 먼저 해결합니다.
- `resolutions`, `overrides`, constraints는 최후의 수단으로만 제안합니다.
- 수정 후에는 OSV/audit/lint/build/기존 test 명령으로 재검증합니다.

## 사용 예

```md
$safe-dependency-fix를 사용해서 이 프로젝트의 패키지 취약점을 분석하고 안전한 수정 계획을 보고해줘.
```

승인 전에는 보고서와 계획까지만 반환합니다.
